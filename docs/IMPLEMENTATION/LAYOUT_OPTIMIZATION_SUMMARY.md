# PNL 曲线图布局优化总结

**优化时间**: 2026-03-22  
**优化内容**: 修复回测图表上半部分被遮盖的问题  
**状态**: ✅ **优化完成**

---

## 🎯 问题诊断

用户反映：回测的 PNL 曲线图上半部分会被遮盖

**根本原因**:
1. 图表高度设置过小（height=500）
2. Legend 位置过靠上（y=0.99），可能遮盖图表数据
3. 缺少合适的 margin 配置，导致图表区域压缩
4. Title 和其他元素占用空间，压低了图表显示区域

---

## ✅ 优化方案

### 1️⃣ **PNL 曲线图（账户净值曲线）**

| 配置项 | 修改前 | 修改后 | 说明 |
|--------|--------|--------|------|
| **height** | 500px | 600px | 增加 100px 高度 |
| **margin.t (top)** | 未设置 | 80px | 预留顶部空间供 title |
| **margin.b (bot)** | 未设置 | 60px | x 轴标签空间 |
| **margin.l (left)** | 未设置 | 70px | y 轴标签空间 |
| **margin.r (right)** | 未设置 | 60px | 右侧 legend 空间 |
| **legend.x** | 0.99 | 0.01 | **移到左侧**，避免遮盖 |
| **legend.y** | 0.99 | 0.95 | 从最顶部降低到 95% |

**修改代码位置**: `GUI_Client/app_v2.py` 第 1023-1033 行

### 2️⃣ **K 线图（数据预览）**

| 配置项 | 修改前 | 修改后 | 说明 |
|--------|--------|--------|------|
| **height** | 600px | 700px | 增加 100px 高度 |
| **margin** | 未设置 | `dict(t=60, b=60, l=70, r=60)` | 添加边距 |

**修改代码位置**: `GUI_Client/app_v2.py` 第 1362-1369 行

## 🔧 技术细节

### League 位置调整
- **从**: `legend=dict(yanchor="top", y=0.99, xanchor="right", x=0.99)`
  - 位置：右上角 (99%, 99%) — 与图表重叠
  
- **到**: `legend=dict(yanchor="top", y=0.95, xanchor="left", x=0.01)`
  - 位置：左上角 (1%, 95%) — 靠近左边框，避免遮盖数据

### Margin 配置解释
```python
margin=dict(
    t=80,   # 顶部：给 title 和图表顶部数据留空间
    b=60,   # 底部：给 x 轴标签留空间
    l=70,   # 左侧：给 y 轴标签留空间
    r=60    # 右侧：给右侧图例和 legend 留空间
)
```

### 高度增加的益处
- 更多像素用于绘制数据，提高清晰度
- 买卖点标记不易拥挤
- 多条线 (策略线、基准线) 更容易区分
